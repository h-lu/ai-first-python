## Vibe Coding 理念回顾

### 传统 vs Vibe Coding

```{mermaid}
flowchart LR
    subgraph S1["传统编程学习"]
        T1["背语法"] --> T2["手写代码"]
        T2 --> T3["调试语法错误"]
        T3 --> T4["期末考试"]
    end
    
    subgraph S2["Vibe Coding"]
        V1["描述需求"] --> V2["AI 生成代码"]
        V2 --> V3["理解和验证"]
        V3 --> V4["迭代优化"]
        V4 --> V5["项目交付"]
    end

    S1 --> S2
```

### 核心理念

::: {.callout-tip}
## Vibe Coding 三原则
1. **描述优于记忆**：Prompt 写得好比语法背得熟更重要
2. **验证优于编写**：能看懂、能改进比从零写更重要
3. **迭代优于完美**：快速试错比一次成功更现实
:::
