15.3 项目优化

优化优先级

常见问题及修复

问题 解决方案
函数过长 拆分为小函数
重复代码 抽取公共函数
魔法数字 使用常量
错误处理缺失 添加 try-except
命名不清 重命名为有意义的名字

优化示例

优化前：

def f(d):
    r = []
    for i in d:
        if i['score'] >= 60:
            r.append(i['name'])
    return r

优化后：

def get_passing_students(students: list[dict]) -> list[str]:
    """获取及格学生的姓名列表。
    
    Args:
        students: 学生字典列表，每个包含 'name' 和 'score' 键
    
    Returns:
        及格学生（分数 >= 60）的姓名列表
    """
    PASSING_SCORE = 60
    
    return [
        student['name'] 
        for student in students 
        if student.get('score', 0) >= PASSING_SCORE
    ]

## 代码审查

### 为什么需要代码审查？

```{mermaid}
flowchart LR
    A["个人开发"] --> B["代码审查"]
    B --> C["发现问题"]
    C --> D["改进代码"]
    D --> E["学习成长"]
```

| 收获 | 说明 |
|-----|------|
| 发现 Bug | 新鲜的眼睛更容易发现问题 |
| 学习技巧 | 看别人怎么写代码 |
| 统一风格 | 团队代码一致性 |
| 知识共享 | 了解不同模块 |

### 审查清单

#### 功能性

- [ ] 代码是否实现了需求描述的功能？
- [ ] 边界情况是否处理？
- [ ] 错误处理是否到位？

#### 可读性

- [ ] 变量/函数命名是否清晰？
- [ ] 是否有必要的注释和文档？
- [ ] 代码结构是否清晰？

#### 安全性

- [ ] API Key 是否硬编码？
- [ ] 用户输入是否验证？
- [ ] 敏感信息是否泄露？

#### 效率

- [ ] 是否有明显的性能问题？
- [ ] 是否有重复代码？
- [ ] 是否有不必要的复杂性？

## 项目优化

### 优化优先级

```{mermaid}
flowchart LR
    A["功能正确"] --> B["错误处理"]
    B --> C["代码清晰"]
    C --> D["性能优化"]
    D --> E["额外功能"]
```

### 常见问题及修复

| 问题 | 解决方案 |
|-----|---------|
| 函数过长 | 拆分为小函数 |
| 重复代码 | 抽取公共函数 |
| 魔法数字 | 使用常量 |
| 错误处理缺失 | 添加 try-except |
| 命名不清 | 重命名为有意义的名字 |

### 优化示例

**优化前**：
```python
def f(d):
    r = []
    for i in d:
        if i['score'] >= 60:
            r.append(i['name'])
    return r
```

**优化后**：
```python
def get_passing_students(students: list[dict]) -> list[str]:
    """获取及格学生的姓名列表。
    
    Args:
        students: 学生字典列表，每个包含 'name' 和 'score' 键
    
    Returns:
        及格学生（分数 >= 60）的姓名列表
    """
    PASSING_SCORE = 60
    
    return [
        student['name'] 
        for student in students 
        if student.get('score', 0) >= PASSING_SCORE
    ]
```
