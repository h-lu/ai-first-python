# 第14周：综合项目开发（上）

## 本周主题
综合运用所学知识，开始期末项目的核心功能开发。

## 学习目标
1. 项目需求分析与设计
2. AI 协同开发流程实践
3. 核心功能模块实现
4. 代码组织与模块化

## 课堂内容

### 第一节：项目启动与设计（45分钟）

1. **需求分析**（15分钟）
   - 确认项目目标和范围
   - 识别核心功能 vs 可选功能
   - 技术可行性评估

2. **架构设计**（15分钟）
   - 模块划分
   - 数据流设计
   - API 调用规划（如适用）

3. **任务分解**（15分钟）
   - 拆分为可实现的小任务
   - 优先级排序
   - 时间估算

### 第二节：开发实践（45分钟）

1. **AI 协同开发**（自主开发）
   - 用 Cursor 生成代码
   - 迭代测试和改进
   - 处理边界情况

2. **教师巡回指导**
   - 解答技术问题
   - 提供设计建议
   - 检查项目进度

## 项目检查点

### 本周必须完成：
- [ ] 项目需求文档
- [ ] 核心功能 50% 实现
- [ ] 至少一个可运行的模块

### 代码规范：
- 函数有文档字符串
- 变量命名有意义
- 错误处理完善

## Prompt 参考

```
我正在开发一个 [项目名称]，需要实现以下功能：
1. [功能1]
2. [功能2]
3. [功能3]

请帮我设计代码结构，包括：
- 需要哪些模块/文件
- 每个模块的主要函数
- 数据如何在模块间传递
```

```
我的项目需要 [具体功能]，请帮我实现：
- 输入：[输入描述]
- 输出：[输出描述]
- 需要处理的边界情况：[边界描述]
```
