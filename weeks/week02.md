# 第2周：计算思维与问题分解

> **课时**：2课时 | **阶段**：环境搭建与计算思维

## 🎯 学习目标

- 理解计算思维的核心概念
- 学会将问题分解为可执行的步骤
- 掌握 Prompt Engineering 基础技巧

---

## 📚 课程内容

### 1. 什么是计算思维？（20分钟）

**四大核心能力**：

| 能力 | 说明 | 示例 |
|-----|------|------|
| **分解** | 把大问题拆成小问题 | 做蛋炒饭 → 打蛋、切葱、炒饭... |
| **模式识别** | 发现规律和重复 | 每道菜都需要"准备食材→加热→调味" |
| **抽象** | 忽略细节，抓住本质 | "烹饪"就是"输入食材，输出菜品" |
| **算法设计** | 设计解决步骤 | 先热锅→放油→放蛋→放饭→调味 |

**讨论**：AI 时代还需要计算思维吗？
- 答案：更需要！AI 负责"写代码"，人负责"想清楚问题"

### 2. 问题分解：从需求到伪代码（30分钟）

**案例分析**：设计一个"成绩统计器"

**第一步：明确需求**
- 输入：一组成绩数据
- 输出：平均分、最高分、最低分、及格率

**第二步：分解步骤**
```
1. 读取成绩数据
2. 计算平均分 = 总分 / 人数
3. 找出最高分
4. 找出最低分
5. 计算及格率 = 及格人数 / 总人数 × 100%
6. 输出结果
```

**第三步：考虑边界情况**
- 如果没有数据怎么办？
- 如果有无效数据（负数、超过100）怎么办？
- 如果有人没考试（缺失值）怎么办？

### 3. Prompt Engineering 基础（40分钟）

**好 Prompt 的要素**：

| 要素 | 说明 | 示例 |
|-----|------|------|
| **明确目标** | 你要什么 | "写一个 Python 函数" |
| **输入输出** | 数据格式 | "输入是成绩列表，输出是字典" |
| **约束条件** | 限制和要求 | "成绩范围 0-100" |
| **示例** | 具体例子 | "例如 [80, 90, 75] → {'avg': 81.67}" |

**Prompt 对比练习**：

❌ 不好的 Prompt：
```
帮我写个成绩统计的程序
```

✅ 好的 Prompt：
```
写一个 Python 函数 analyze_scores(scores)：
- 输入：成绩列表，如 [85, 92, 78, 60, 45]
- 输出：字典，包含 avg（平均分）、max、min、pass_rate（及格率，60分及格）
- 要求：如果列表为空，返回 None
- 示例：analyze_scores([80, 90, 70]) → {'avg': 80.0, 'max': 90, 'min': 70, 'pass_rate': 100.0}
```

### 4. 实战：用自然语言实现计算器（30分钟）

**课堂练习**：

与 AI 对话，实现一个命令行计算器：
- 支持加减乘除
- 用户输入表达式，输出结果
- 输入 "quit" 退出

**引导问题**：
1. 如何让 AI 理解"支持加减乘除"？
2. 如何处理用户输入错误？
3. 如何让程序循环运行直到用户退出？

## 🎓 Prompt 模板参考

```
我需要一个 Python 程序/函数来 [做什么]。

输入：[描述输入格式和示例]
输出：[描述输出格式和示例]

要求：
1. [具体要求1]
2. [具体要求2]
3. [边界情况处理]

示例：
输入：[具体示例]
输出：[期望结果]
```

## 💡 教学提示

- 在 Prompt 练习中，让学生互相比较不同 Prompt 的效果
- 强调"边界情况"——这是 AI 经常忽略的地方
- 鼓励学生"迭代"：第一次 Prompt 不完美是正常的
