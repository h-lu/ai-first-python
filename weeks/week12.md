# 第12周：LLM 应用开发

## 本周主题
深入 LLM API 使用，掌握 Prompt Engineering，构建实用的 AI 应用。

## 学习目标
1. 掌握多轮对话实现
2. 理解 System Prompt 设计
3. 学习 Prompt Engineering 技巧
4. 构建有特定功能的聊天机器人

## 课堂内容

### 第一节：LLM API 进阶（45分钟）

1. **多轮对话实现**（15分钟）
   - messages 列表的作用
   - 对话历史管理
   - 上下文窗口限制

2. **System Prompt 设计**（15分钟）
   - 什么是 System Prompt
   - 设定 AI 角色和行为
   - 示例：代码助手、翻译器、创意写作

3. **流式输出**（15分钟）
   - 为什么需要流式输出
   - stream=True 参数
   - 逐字显示效果

### 第二节：Prompt Engineering（45分钟）

1. **Few-shot Prompting**（15分钟）
   - 通过示例引导输出格式
   - 案例：情感分析、信息提取

2. **思维链提示**（15分钟）
   - Chain of Thought (CoT)
   - 让 AI "说出思考过程"
   - 适用场景：数学、逻辑推理

3. **实战：构建聊天机器人**（15分钟）
   - 定义角色和能力边界
   - 处理用户输入
   - 格式化输出

## 课堂练习

### 练习 1：多轮对话
实现一个保持上下文的对话程序：
- 维护对话历史
- 用户输入 "exit" 退出
- 支持查看对话历史

### 练习 2：自定义 AI 助手
创建一个有特定人设的 AI：
- 定义 System Prompt
- 添加 Few-shot 示例
- 测试不同场景

## Prompt 工程模板

```
## System Prompt 模板
你是一个[角色]，专门帮助用户[做什么]。

你的特点：
- [特点1]
- [特点2]

回答规则：
- [规则1]
- [规则2]

## Few-shot 模板
以下是一些示例：

用户：[输入1]
助手：[输出1]

用户：[输入2]
助手：[输出2]
```
